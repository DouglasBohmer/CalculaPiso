Parte 1: O arquivo pom.xml (O que é e onde fica?)
Sua pergunta indica que seu projeto atual no Eclipse provavelmente não é um projeto "Maven". O Maven é uma ferramenta que gerencia as dependências (as bibliotecas, como o Selenium) do seu projeto automaticamente. O arquivo pom.xml é o "coração" de um projeto Maven, onde você lista quais bibliotecas e versões precisa.

Em vez de criar o arquivo pom.xml do zero, o jeito mais fácil é converter seu projeto existente para um projeto Maven dentro do Eclipse.

Siga estes passos:

Backup (Segurança): Antes de tudo, faça uma cópia de segurança da pasta do seu projeto. É sempre bom ter um backup antes de fazer mudanças estruturais.

Converter o Projeto:

No Eclipse, na aba "Package Explorer" (onde ficam seus arquivos), clique com o botão direito no seu projeto (DownloadCVM).

Vá até o menu Configure.

Clique na opção "Convert to Maven Project...".

Criar o pom.xml:

Uma janela "Create new POM" vai aparecer. Você só precisa preencher os campos básicos. Eles servem para identificar seu projeto. Pode usar estes valores:

Group Id: br.com.cvm

Artifact Id: downloader-cvm

Version: 0.0.1-SNAPSHOT

Packaging: jar

Clique em Finish.

Editar o pom.xml:

O Eclipse vai criar o arquivo pom.xml na raiz do seu projeto. Ele também pode criar uma estrutura de pastas como src/main/java. Seus arquivos .java devem ficar dentro dessa pasta.

Abra o arquivo pom.xml. No editor que aparecer, clique na aba "pom.xml" na parte de baixo para ver o código fonte.

Agora, adicione o bloco <dependencies> que eu mencionei, dentro da tag <project>:

XML

<project ...>
  ... (outras tags como groupId, artifactId) ...

  <dependencies>
    <dependency>
        <groupId>org.seleniumhq.selenium</groupId>
        <artifactId>selenium-java</artifactId>
        <version>4.21.0</version>
    </dependency>

    <dependency>
        <groupId>org.jsoup</groupId>
        <artifactId>jsoup</artifactId>
        <version>1.17.2</version>
    </dependency>
  </dependencies>

</project>
Salvar: Salve o arquivo pom.xml. O Eclipse vai automaticamente baixar as bibliotecas do Selenium e do Jsoup e adicioná-las ao seu projeto. Você verá uma barra de progresso no canto inferior direito.

Parte 2: A Questão da Versão em 2 Micros Diferentes
Essa é uma excelente pergunta e é exatamente o problema que o Maven resolve.

1. A Versão das Bibliotecas (Selenium, Jsoup):

É aqui que a mágica do pom.xml acontece. Este arquivo faz parte do seu projeto. Quando você copiar a pasta do seu projeto para o outro computador (ou usar um sistema como Git/Dropbox), o pom.xml vai junto.

Ao abrir o projeto no Eclipse do outro computador, o Maven vai ler o pom.xml e automaticamente baixar as mesmas versões das bibliotecas (selenium-java versão 4.21.0, etc.).

Resultado: Isso garante que o seu código Java funcione exatamente igual nos dois computadores, sem que você precise copiar arquivos .jar manualmente.

2. A Versão do ChromeDriver (O Ponto de Atenção Manual):

Esta é a parte que exige uma pequena ação manual em cada máquina, mas apenas uma vez.

O arquivo chromedriver.exe precisa corresponder à versão do navegador Google Chrome instalado em CADA computador. É muito provável que seu Chrome de casa e do trabalho não estejam na mesma versão exata.

O que fazer:

No Computador 1:

Abra o Chrome, vá em "Ajuda" -> "Sobre o Google Chrome" e veja a versão (ex: 125.0.6422.142).

Baixe o chromedriver.exe para essa versão exata. -> https://googlechromelabs.github.io/chrome-for-testing/

Coloque o arquivo em C:\WebDriver\chromedriver.exe.

No Computador 2:

Faça a mesma coisa. Abra o Chrome, veja a versão (pode ser diferente, ex: 125.0.6422.170).

Baixe o chromedriver.exe para essa outra versão.

Coloque o arquivo na mesma pasta C:\WebDriver\chromedriver.exe.

O seu código Java (System.setProperty("webdriver.chrome.driver", "C:\\WebDriver\\chromedriver.exe");) vai funcionar perfeitamente nos dois micros, pois ele sempre procurará o driver nesse caminho padrão que definimos.

Resumindo:

pom.xml: Cuida das versões das bibliotecas Java (Selenium) automaticamente em qualquer máquina.

chromedriver.exe: Você precisa garantir que o arquivo em C:\WebDriver\ seja o correto para a versão do Chrome daquela máquina específica.
